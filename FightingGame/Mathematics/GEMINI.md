This directory (`/Users/aramsamifanni/obsidian/FightingGame/Mathematics`) serves as a mathematical knowledge base for game development, specifically shaders and physics. 

## File Structure & Map Guide
- **Core Concepts:** Files are typically named after the core concept (e.g., `Matrix.md`, `Vector Calculus.md`, `GLSL Shaders.md`).
- **Shader Math:** Notes like `UV Normalization.md`, `Space Repetition.md`, `Domain Warping.md`, `Smoothstep Interpolation.md`, and `Simplex Noise.md` detail specific GLSL techniques.
- **Interlinking:** Always use double-bracket Obsidian links `[[Note Name]]` to connect related concepts (e.g., linking `[[Matrices]]` inside `Affine Transformations.md`).

## Content Style Guide
1. **Tags:** Place tags at the top of the file (e.g., `#math`, `#shaders`, `#physics`).
2. **Headings:** Use clear H1 `#` for titles and H2 `##` or H3 `###` for sub-sections.
3. **Definitions First:** Start notes with a clear, concise definition of the mathematical concept or function.
4. **Code Blocks:** Use explicitly typed code blocks (e.g., ` ```glsl ` for shader code, ` ```math ` or `$$` for equations).
5. **Formatting:** Use blockquotes (`>`) for important callouts or structural explanations.
6. **Brevity:** Keep notes focused and modular. If a note gets too long, split it and link them.
